The Voice UI Kit ships with a default theme that can be used as-is or modified to your liking.

It is implemented using [Tailwind CSS](https://tailwindcss.com/) V4 and built to be compatible with a developer's own styling or Tailwind implementation.

- `@pipecat-ai/voice-ui-kit/styles.css` - The default theme, prefixed with `vkui:`
- `@pipecat-ai/voice-ui-kit/styles.scoped.css` - The default theme, scoped to a `.vkui-root` class
- Dark mode support (via `:root.vkui:dark`)
- ShadUI compatibility (via `@theme static`)

## Adding styles

The Voice UI Kit CSS can be imported from the `@pipecat-ai/voice-ui-kit/styles` package.

The CSS file can be imported in your application CSS file (e.g., `global.css`) or directly in your JavaScript code (e.g., `layout.tsx`).

```css title="global.css"
@import "@pipecat-ai/voice-ui-kit/styles";

/* ... */
```

If you're using Tailwind, you can import it alongside your own custom styles:

```css title="global.css"
@import "tailwindcss";
@import "tw-animate-css";

@import "@pipecat-ai/voice-ui-kit/styles";

/* ... */
```

You can also include the styles in your application code:

```tsx title="layout.tsx"
import "./globals.css";

import "@pipecat-ai/voice-ui-kit/styles";

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

### Production-ready builds

Bundled styles are left unoptimized, with all utility classes left inlined. This allows developers to choose a build and optimization strategy that works best for them.

For example, in production, it's recommended to apply minification and other post-processing to the styles at build time. Most modern frameworks (such as Vite, Next.js, etc.) have out-of-the-box support for this.

## Prefixing and scoping

Voice UI Kit variables and utilities are prefixed with a `vkui` namespace. This means you can use them without worrying about conflicts with your own styles.

```html
<div class="vkui:flex">...</div>
```

By design, the styles export is left unscoped. This means using the CSS import alongside your own project's Tailwind theme will apply to Voice UI Kit components as well.

While this can be preferable in some cases, sometimes it's not ideal. The Voice UI Kit bundle includes a fully scoped CSS file that can be used instead:

```css title="global.css"
@import "@pipecat-ai/voice-ui-kit/styles.scoped";

/* Your application CSS file */
```

The Voice UI Kit theme will now only apply when wrapped in a `.vkui-root` class, and declarations from your own Tailwind theme will not be applied.

```html
<div class="vkui-root">
  <span class="vkui:flex">✅ Scoped correctly</span>
</div>

<div>
  <span class="vkui:flex">❌ Not scoped</span>
</div>
```

## Modifying styles

The Voice UI Kit theme is fully customizable and can be aligned with your application's design system.

```css title="global.css"
@import "@pipecat-ai/voice-ui-kit/styles";

:root {
  /* Light mode overrides */
  --vkui-color-background: oklch(1 0 0);
  --vkui-color-primary: var(--color-blue-500);
}

.vkui\:dark {
  /* Dark mode overrides */
  --vkui-color-background: oklch(20.5% 0.004 285.885);
  --vkui-color-primary: var(--color-blue-400);
}
```

## Dark mode

The Voice UI Kit theme supports dark mode, using a `:root.vkui:dark` selector:

```html
<html class="vkui:dark">
  <body>
    ...
  </body>
</html>
```

### Programmatic dark mode

You can also toggle dark mode programmatically by adding or removing the `vkui:dark` class from your HTML element:

```javascript
// Enable dark mode
document.documentElement.classList.add("vkui:dark");

// Disable dark mode
document.documentElement.classList.remove("vkui:dark");

// Toggle dark mode
document.documentElement.classList.toggle("vkui:dark");
```
